Technology stack

  • Client-side MVC framework: Backbone.js along with Marionette.js
  • Modular Dependencies: Require.js
  • CSS Libraries: Twitter Bootstrap Responsive, CSS Pre-processor: SASS
  • Testing framework: Mocha
  • Server: Node.js, Middleware: Express.js
  • Version control: Git
  • Deployment Environment: Appfog Cloud-based PAAS provider

My Approach

  • I wanted to showcase how I have used Backbone Model and Collection objects along with Marionette Views to create the demo.
  • The Model object that I have created for this test is called "ProgressMappingModel.js", with progress bar specific functions for incrementing and decrementing progress.
  • The View object handles the events and invokes Model functions and reflects the output back into the DOM.
  • The presence of Multiple progress bars in the demo are handled by having a Collection of "ProgressMappingModel" Model objects and proved to be useful due to convenient functions provided by Collections API.
  • The separation of Presentation logic in the Model helped me perform the Tests.
